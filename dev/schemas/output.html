Runtime timestamp

Time at which ONM algorithm was executed

Runtime timestamp

  • is required

  • Type: string

  • cannot be null

  • defined in: Outputs

Runtime timestamp Type

string

Runtime timestamp Constraints

date time: the string must be a date time string, according to RFC 3339, section 5.6

Simulation time steps

The actual timesteps, either in datetime strings, or as numbers

Simulation time steps

  • is required

  • Type: an array of the following:string or number (Details)

  • cannot be null

  • defined in: Outputs

Simulation time steps Type

an array of the following:string or number (Details)

Powerflow output

The schema for the powerflow outputs from [optimize_dispatch](@ref optimize_dispatch), which include real and reactive power setpoints at generation objects, which include diesel generators (generator), substations/grid/feeder (voltage_source), solar PV (solar), or energy storage (storage), and voltage magnitudes at buses. All values are in SI units.

Powerflow output

  • is required

  • Type: object[] (Details)

  • cannot be null

  • defined in: Outputs

Powerflow output Type

object[] (Details)

Events

A list of events, ordered (i.e., later events will overwrite earlier ones if there is a conflict), that can either be a switching action, or a fault.

Events

  • is optional

  • Type: object[] (Details)

  • cannot be null

  • defined in: Outputs

Events Type

object[] (Details)

Events Constraints

minimum number of items: the minimum number of items for this array is: 0

Events Examples

[
  {
    "timestep": 1,
    "event_type": "switch",
    "affected_asset": "line.switch_name",
    "event_data": {
      "type": "breaker",
      "state": "open",
      "dispatchable": false,
      "status": 1
    }
  }
]
